# 节流

有关节流的解释，你可以在这里在新窗口打开 找到。

简单来说就是，保证每 X 毫秒去执行一次函数。例如输入事件，throttle 函数会在用户输入的过程中以设定的时间间隔去执行函数。这与我们上一节提到的 debounce 有很大的区别。

注意

VueRequest 的节流是使用 lodash在新窗口打开 提供的 throttle在新窗口打开 实现的，leading 和 trailing 选项默认为 true，因此，在第一个 throttle 的执行间隔中可能会触发两次请求

如下所示，你只需要提供一个 throttleInterval 来告诉我们节流的执行间隔。这样用户在输入信息时，每隔 300 毫秒 我们就会发起一次请求。

import { defineComponent } from 'vue';
import { useRequest } from 'vue-request';
import Mock from 'mockjs';

function testService() {
  return new Promise<{ array: string[] }>((resolve) => {
    setTimeout(() => {
      resolve(
        Mock.mock({
          'array|3-5': ['@name'],
        }),
      );
    }, 1000);
  });
}

export default defineComponent({
  name: 'App',
  setup() {
    const { data, loading, run } = useRequest(testService, {
      throttleInterval: 300,
      manual: true,
    });
    return () => (
      <div>
        <input placeholder="Enter the name" type="text" onInput={() => run()} />
        <br />
        <ul>
          {loading.value
            ? 'loading...'
            : data.value?.array?.map((item) => <li key={item}>{item}</li>)}
        </ul>
      </div>
    );
  },
});
